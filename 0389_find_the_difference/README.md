# 0389 — Find the Difference

> LeetCode #389 · Difficulty: Easy
> **Categories:** Bit Manipulation, Hash Table, String, Counting

---

## Problem Statement

You are given two strings `s` and `t`.

String `t` is generated by random shuffling string `s` and then add one more letter at a random position.

Return the letter that was added to `t`.

**Example 1:**

```
Input: s = "abcd", t = "abcde"
Output: "e"
Explanation: 'e' is the letter that was added.
```

**Example 2:**

```
Input: s = "", t = "y"
Output: "y"
```

**Constraints:**

- `0 <= s.length <= 1000`
- `t.length == s.length + 1`
- `s` and `t` consist of lowercase English letters.

## Approach 1 — Frequency Count (26-array)

### Intuition

`t` is `s` shuffled plus one extra letter. Add 1 for every letter of `t` and subtract 1 for every letter of `s`: shared letters cancel to 0, and only the added letter ends with a positive count.

### Algorithm

1. `count[t[i]-'a']++` for all `i`.
2. `count[s[i]-'a']--` for all `i`.
3. Return the single letter whose count is `> 0`.

### Complexity

- **Time:** O(n) — two linear passes.
- **Space:** O(1) — 26 counters.

### Code

```go
func countArray(s string, t string) byte {
	var count [26]int
	for i := 0; i < len(t); i++ {
		count[t[i]-'a']++ // t is one char longer; tally it up
	}
	for i := 0; i < len(s); i++ {
		count[s[i]-'a']-- // cancel out every letter that also appears in s
	}
	for i := 0; i < 26; i++ {
		if count[i] > 0 { // the leftover positive count is the added letter
			return byte('a' + i)
		}
	}
	return 0 // unreachable given the problem guarantees
}
```

### Dry Run

Input `s = "abcd"`, `t = "abcde"`:

| pass | operation | count changes |
|------|-----------|---------------|
| t | a,b,c,d,e each +1 | a=1,b=1,c=1,d=1,e=1 |
| s | a,b,c,d each −1 | a=0,b=0,c=0,d=0,e=1 |
| scan | first count>0 | index of `e` → **return 'e'** |

Answer: `e`.

## Approach 2 — XOR Bit Manipulation (Optimal)

### Intuition

XOR is its own inverse (`x ^ x == 0`) and is commutative/associative. Across the combined stream of `s` and `t`, every shared letter appears an even number of times and cancels to 0; the one extra letter appears an odd number of times and survives as the running XOR.

### Algorithm

1. `acc = 0`.
2. XOR `acc` with every byte of `s` and every byte of `t`.
3. Return `acc`.

### Complexity

- **Time:** O(n) — one pass over each string.
- **Space:** O(1) — a single accumulator byte.

### Code

```go
func xorBits(s string, t string) byte {
	var acc byte // XOR accumulator
	for i := 0; i < len(s); i++ {
		acc ^= s[i] // fold in each char of s
	}
	for i := 0; i < len(t); i++ {
		acc ^= t[i] // fold in each char of t; matched pairs cancel
	}
	return acc // odd one out remains
}
```

### Dry Run

Input `s = "abcd"`, `t = "abcde"`. Using char codes (a=97…e=101):

| step | char | acc (running XOR) |
|------|------|-------------------|
| s | a | 97 |
| s | b | 97^98 |
| s | c | 97^98^99 |
| s | d | 97^98^99^100 |
| t | a | …^97 → a cancels |
| t | b,c,d | each cancels its s twin |
| t | e | acc = 101 = 'e' |

Answer: `e`.

## Approach 3 — ASCII Sum Difference

### Intuition

Every shared letter contributes its ASCII code to both sums, so those contributions cancel in `(sum of t) − (sum of s)`. What remains is exactly the extra character's code. Pure integer arithmetic, no tables.

### Algorithm

1. `diff = 0`. Add each `t[i]` to `diff`; subtract each `s[i]` from `diff`.
2. Return `byte(diff)`.

### Complexity

- **Time:** O(n) — one pass over each string.
- **Space:** O(1) — a single integer accumulator.

### Code

```go
func sumDiff(s string, t string) byte {
	diff := 0 // (sum of t codes) − (sum of s codes)
	for i := 0; i < len(t); i++ {
		diff += int(t[i]) // add every code of t
	}
	for i := 0; i < len(s); i++ {
		diff -= int(s[i]) // remove every code of s
	}
	return byte(diff) // leftover = code of the extra letter
}
```

### Dry Run

Input `s = "abcd"`, `t = "abcde"`:

| pass | running diff |
|------|--------------|
| +t: a,b,c,d,e | 97+98+99+100+101 = 495 |
| −s: a,b,c,d | 495 − (97+98+99+100) = 495 − 394 = 101 |
| result | `byte(101)` = 'e' |

Answer: `e`.

---

## Key Takeaways

- **XOR cancels pairs.** The "find the single odd element among duplicated pairs" pattern (also LeetCode #136 Single Number) is the go-to for O(1)-space uniqueness.
- **Counting generalises; XOR/sum are the two extra letters is exactly one.** Frequency counting still works if more than one letter is added; XOR and sum only isolate a *single* extra element.
- **Sum trick is elegant but overflow-prone** on very long strings or wide alphabets — use `int`, not `byte`, for the accumulator.
