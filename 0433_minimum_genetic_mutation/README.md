# 0433 — Minimum Genetic Mutation

> LeetCode #433 · Difficulty: Medium
> **Categories:** Hash Table, String, Breadth-First Search

---

## Problem Statement

A gene string can be represented by an 8-character long string, with choices from `'A'`, `'C'`, `'G'`, and `'T'`.

Suppose we need to investigate a mutation from a gene string `startGene` to a gene string `endGene` where one mutation is defined as one single character changed in the gene string.

- For example, `"AACCGGTT" --> "AACCGGTA"` is one mutation.

There is also a gene bank `bank` that records all the valid gene mutations. A gene must be in `bank` to make it a valid gene string.

Given the two gene strings `startGene` and `endGene` and the gene bank `bank`, return *the minimum number of mutations needed to mutate from* `startGene` *to* `endGene`. If there is no such a mutation, return `-1`.

Note that the starting point is assumed to be valid, so it might not be included in the bank.

**Example 1:**

```
Input: startGene = "AACCGGTT", endGene = "AACCGGTA", bank = ["AACCGGTA"]
Output: 1
```

**Example 2:**

```
Input: startGene = "AACCGGTT", endGene = "AAACGGTA", bank = ["AACCGGTA","AACCGCTA","AAACGGTA"]
Output: 2
```

**Example 3:**

```
Input: startGene = "AAAAACCC", endGene = "AACCCCCC", bank = ["AAAACCCC","AAACCCCC","AACCCCCC"]
Output: 3
```

**Constraints:**

- `0 <= bank.length <= 10`
- `startGene.length == endGene.length == bank[i].length == 8`
- `startGene`, `endGene`, and `bank[i]` consist of only the characters `['A', 'C', 'G', 'T']`.

## Approach 1 — Breadth-First Search (Shortest Path)

### Intuition

Treat each gene as a node in a graph; put an edge between two genes when they differ in exactly one character (one mutation). The minimum number of mutations from `startGene` to `endGene` is then the length of the **shortest path** between them. For an **unweighted** graph, shortest path = **BFS**: explore everything reachable in 1 mutation, then 2, then 3, … The first time `endGene` appears, the current level number is the answer. Marking a gene visited (removing it from the bank set) the moment we enqueue it prevents cycles and re-work.

### Algorithm

1. Put `bank` into a set `valid`. If `endGene ∉ valid`, return `-1` (unreachable).
2. BFS from `startGene`, tracking `steps` = mutations used to reach the current level.
3. To expand a gene, try every position `i` × every letter `c ∈ {A,C,G,T}`; a neighbour is any resulting string still present in `valid`.
4. When a neighbour is found, delete it from `valid` (mark visited) and add it to the next level.
5. When `endGene` is popped, return `steps`. If BFS drains, return `-1`.

### Complexity

- **Time:** O(N · L) — each of ≤ N genes is expanded once by trying `L` positions × 4 letters (constant), and each candidate string costs O(L) to build.
- **Space:** O(N · L) — the bank set and BFS queue hold up to N genes of length L.

### Code

```go
func bfs(startGene, endGene string, bank []string) int {
	valid := make(map[string]bool, len(bank))
	for _, g := range bank {
		valid[g] = true
	}
	if !valid[endGene] {
		return -1 // target not in bank → impossible
	}
	choices := []byte{'A', 'C', 'G', 'T'}
	queue := []string{startGene}
	steps := 0
	for len(queue) > 0 {
		next := []string{}
		for _, gene := range queue {
			if gene == endGene {
				return steps // shortest distance found
			}
			b := []byte(gene)
			for i := 0; i < len(b); i++ {
				original := b[i]
				for _, c := range choices {
					if c == original {
						continue
					}
					b[i] = c
					cand := string(b)
					if valid[cand] {
						valid[cand] = false // mark visited
						next = append(next, cand)
					}
				}
				b[i] = original // restore
			}
		}
		queue = next
		steps++
	}
	return -1
}
```

### Dry Run

Example 1: `start = "AACCGGTT"`, `end = "AACCGGTA"`, `bank = ["AACCGGTA"]`, so `valid = {AACCGGTA}`.

| Level (steps) | queue | expand | action |
|---------------|-------|--------|--------|
| 0 | `["AACCGGTT"]` | pop `AACCGGTT` (≠ end); mutate position 7 `T→A` gives `AACCGGTA` ∈ valid | mark `AACCGGTA` visited, add to next |
| 1 | `["AACCGGTA"]` | pop `AACCGGTA` == end | **return steps = 1** |

Output: `1` ✔ (Example 2 → `2`, Example 3 → `3`, empty bank → `-1`.)

## Approach 2 — Bidirectional BFS (Optimal)

### Intuition

A single BFS explores a ball whose size grows exponentially with the radius `d`. If we search from **both** ends and meet in the middle, each side only needs radius ~`d/2`, shrinking the explored volume from roughly `b^d` to `2·b^(d/2)` — a big win when the path is long. The key optimisation: at each step **expand whichever frontier is smaller**, so we always branch from the fewest nodes. As soon as a newly generated neighbour is found in the *opposite* frontier, the two half-paths join and the total mutation count is `steps + 1`.

### Algorithm

1. `bank → valid` set; if `endGene ∉ valid`, return `-1`.
2. Keep two frontier sets `begin = {start}` and `end = {end}`, plus a `visited` set; `steps = 0`.
3. While both frontiers are non-empty: swap so `begin` is the smaller side, then expand it. For each gene, generate every valid one-char neighbour:
   - if the neighbour is in `end` → return `steps + 1` (frontiers meet).
   - else if unvisited → mark visited and add to the next layer.
4. Replace `begin` with its next layer; `steps++`.
5. If the loop ends without meeting, return `-1`.

### Complexity

- **Time:** O(N · L) worst case, but visits far fewer nodes than one-directional BFS (≈ square root of the search volume).
- **Space:** O(N · L) — two frontier sets plus the visited/bank sets.

### Code

```go
func bidirectionalBFS(startGene, endGene string, bank []string) int {
	valid := make(map[string]bool, len(bank))
	for _, g := range bank {
		valid[g] = true
	}
	if !valid[endGene] {
		return -1
	}
	if startGene == endGene {
		return 0
	}
	choices := []byte{'A', 'C', 'G', 'T'}
	begin := map[string]bool{startGene: true}
	end := map[string]bool{endGene: true}
	visited := map[string]bool{startGene: true, endGene: true}
	steps := 0
	for len(begin) > 0 && len(end) > 0 {
		if len(begin) > len(end) {
			begin, end = end, begin // always expand the smaller frontier
		}
		next := map[string]bool{}
		for gene := range begin {
			b := []byte(gene)
			for i := 0; i < len(b); i++ {
				original := b[i]
				for _, c := range choices {
					if c == original {
						continue
					}
					b[i] = c
					cand := string(b)
					if end[cand] {
						return steps + 1 // frontiers meet
					}
					if valid[cand] && !visited[cand] {
						visited[cand] = true
						next[cand] = true
					}
				}
				b[i] = original
			}
		}
		begin = next
		steps++
	}
	return -1
}
```

### Dry Run

Example 1: `start = "AACCGGTT"`, `end = "AACCGGTA"`, `valid = {AACCGGTA}`.

| steps | begin (smaller) | end | expand begin | result |
|-------|-----------------|-----|--------------|--------|
| 0 | `{AACCGGTT}` | `{AACCGGTA}` | mutate `AACCGGTT` pos 7 `T→A` → `AACCGGTA`, which is in `end` | **return steps+1 = 1** |

Output: `1` ✔ — the two single-node frontiers meet after one expansion.

## Approach 3 — DFS with Depth Memo

### Intuition

The same graph, explored depth-first. Plain DFS can loop forever and does not naturally produce the *shortest* path, so we add two safeguards: memoise the **minimum depth** at which each gene has been reached, and cut any branch that revisits a gene at an equal-or-greater depth (it can't improve the answer). This approach is here for contrast — it demonstrates why BFS is the natural fit: BFS gets the shortest path for free, whereas DFS needs explicit depth bookkeeping to achieve the same.

### Algorithm

1. `bank → valid` set; if `endGene ∉ valid`, return `-1`.
2. Recurse from `startGene` at depth `0`. At each gene:
   - if `gene == endGene`, update `best` with the smaller depth and return.
   - prune if `bestDepth[gene] ≤ depth` (already reached at least this cheaply).
   - otherwise record `bestDepth[gene] = depth` and recurse into every valid one-char neighbour at `depth + 1`.
3. Return `best` (or `-1` if never reached).

### Complexity

- **Time:** O(N · L) amortised — the depth memo means a gene is meaningfully expanded only when reached more cheaply than before.
- **Space:** O(N · L) for the sets/memo plus O(N) recursion depth.

### Code

```go
func dfs(startGene, endGene string, bank []string) int {
	valid := make(map[string]bool, len(bank))
	for _, g := range bank {
		valid[g] = true
	}
	if !valid[endGene] {
		return -1
	}
	choices := []byte{'A', 'C', 'G', 'T'}
	bestDepth := map[string]int{}
	best := -1

	var explore func(gene string, depth int)
	explore = func(gene string, depth int) {
		if gene == endGene {
			if best == -1 || depth < best {
				best = depth
			}
			return
		}
		if d, seen := bestDepth[gene]; seen && d <= depth {
			return // no improvement possible from here
		}
		bestDepth[gene] = depth
		b := []byte(gene)
		for i := 0; i < len(b); i++ {
			original := b[i]
			for _, c := range choices {
				if c == original {
					continue
				}
				b[i] = c
				cand := string(b)
				if valid[cand] {
					explore(cand, depth+1)
				}
			}
			b[i] = original
		}
	}
	explore(startGene, 0)
	return best
}
```

### Dry Run

Example 1: `start = "AACCGGTT"`, `end = "AACCGGTA"`, `valid = {AACCGGTA}`.

| Call | gene | depth | action |
|------|------|-------|--------|
| 1 | `AACCGGTT` | 0 | ≠ end; not memoised → set `bestDepth[AACCGGTT]=0`; mutate pos 7 `T→A` → `AACCGGTA` ∈ valid → recurse |
| 2 | `AACCGGTA` | 1 | == end → `best = 1`, return |
| — | (other mutations of `AACCGGTT`) | 1 | none are in `valid` → skipped |

Result: `best = 1` ✔

## Key Takeaways

- **"Fewest one-step changes to transform X into Y" = shortest path in an implicit graph.** Nodes are states, edges are single legal moves; BFS gives the minimum in unweighted graphs. This template also solves Word Ladder (#127) and Open the Lock (#752).
- **Generate neighbours on the fly** rather than precomputing the whole graph: for each position, try each of the 4 letters and keep only candidates present in the bank set. No adjacency list needed.
- **A hash set is the workhorse** — it both validates candidates and doubles as the visited set (delete on enqueue).
- **Bidirectional BFS** roughly square-roots the number of explored nodes; the trick that makes it correct *and* fast is always expanding the smaller frontier.
- **BFS vs DFS for shortest path:** reach for BFS. DFS can be forced to work with a depth memo, but the extra bookkeeping is exactly the guarantee BFS gives you for free.
